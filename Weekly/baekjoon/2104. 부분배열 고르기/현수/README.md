# 2104. 부분배열 고르기

<h2>느낀점</h2>
입력과 제한 시간을 처음 봤을 때 단순한 접근으로는 불가능할 것이라고 생각했다. 그래서 생각해보던 중, 예전에 대충 보고 지나갔었던 세그먼트 트리가 생각나서 다시 찾아보면서 이를 적용했다. 이유는 이것이 부분합을 구하는 가장 빠른 방법이었기에 이를 적용하였다. 또한 종만북에서 봤던 비슷한 문제를 떠올리며 풀 수 있었다. 분할정복에 대한 감이 아직은 부족한 것 같다. 유용한 자료구조를 다시 찾아볼 수 있었고, 분할 정복을 느껴볼 수 있었던 좋은 문제인 것 같다.
<hr>

<h2>풀이</h2>
<ol type="I">
    <li> 세그먼트 트리 작성 </li>
    <li> 분할 정복 : mid를 기준으로 왼쪽, 오른쪽 구한 후 mid를 걸치는 구간의 값도 계산 </li>
    <li> 걸치는 구간의 최대 값은 단순 그리디 탐색으로, 왼쪽 탐색 끝과 오른쪽 탐색 끝의 값 중 더 큰 값을 기준으로 확장해나감. </li>
</ol>
