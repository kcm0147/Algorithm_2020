# 17825. 주사위 윷놀이

<h2>느낀점</h2>
>많이 어려웠다. 3일동안 고민하고 고민해서 모든 조건을 다 찾아냈는데, TC3번이 통과하지 않았다.. 분명 다른 경로의 모든 지점까지 고려했는데 왜지/? 결국 풀이방법을 싹 다 갈아엎고 다시 풀었다. 다른 사람의 풀이를 보며 이렇게도 되는데 왜 내꺼는 안되지라는 생각밖에 안들었다.. 간단하게 짜는게 알 수 없는 실수를 줄여주는 최선의 방법인 것 같다. (파이썬 식으로 생각하는 것도 굉장히 도움이 많이 되었다.)
<hr>

<h2>풀이</h2>
<ol type="I">
    <li> 보드판을 5가지의 경로로 나눠서 생각한다. </li>
    <li> 각 말의 정보를 배열로 담아 저장한다. 이 때 나는 처음에 구조체로 구현하였는데, 이후 코드가 복잡해지는 원인이 되었다.</li>
    <li> 먼저 현재 주사위 정보만큼 말의 위치를 계산하였다. 이후 경로까지 변경해준다. </li>
    <li> 도착지를 넘어가면 점수를 0으로 두고, 끝나는 지점을 각 경로의 마지막 위치로 계산하였다. </li>
</ol>
<hr>

<h2>실수</h2>
>구조체에 각 말판의 경로도 포인터로 추가하여 생각의 정도가 복잡해지는 결과를 초래.. 그 전에는 레퍼런스와 셋, 경로벡터를 다 구현해서 시간도 오래걸리고 복잡해지기도 했었다.. 
>배열로 단순화하면서 시간을 줄일 수 있었으나, 여러 부분적인 조건들을 추가하면서 이래저래 서로 얽히고 섥히면서 굉장히 시간을 많이 쏟아야만 했다..
>결국 3일동안 짰던 코드를 싹 다 엎고, 정답자의 풀이를 참고하며 정말 단순한 생각부터 구현하며 짰다. -> 파이썬으로 구현하는 것처럼 진행하는 생각을 기르는게 매우 유용한 것 같다!!
