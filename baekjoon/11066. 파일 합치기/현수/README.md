# 11066. 파일 합치기

<h2>느낀점</h2>
행렬연쇄곱 풀이와 비슷한 느낌이었다. 2차원 배열에 바텀업 방식으로 열 우선 탐색을 했는데, 첫번째 케이스는 맞지만 두번째가 너무 크게 나왔다. 직접 0~6까지 하나하나 구해봤는데 일치했지만 답이 이상하게 나왔다.. 아무리 생각해도 될 거 같은데 안되길래 그냥 일차원 배열로 풀었다. 
    
<hr>
<h2>풀이</h2>
<ol type="I">
    <li> 파일을 합치는 것은 두 개의 부분 문제를 푼 것들을 합치는 것과 같다. 따라서 두 부분으로 나누어서 이들의 최소합을 구해준다. </li>
    <li> dp(start, i) + dp(i + 1, end) 로 최소값을 갱신하는데, 최종적으로 나눠진 두 부분의 범위안에 들어가는 모든 수의 합이 더해져야 한다 </li>
    <li> 최적화를 위해 부분합을 미리 구해놓고 이를 활용한다. </li>
</ol>
<hr>
<h2> 실수 </h2>
 2차원 배열로 생각했을 때는 왜 안되는지 모르겠다. 1차원보다 로직이 복잡해지면서 내가 알아채지 못한 실수의 여지가 늘어난 것 같다..
